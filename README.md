# 14th-collaborate-practice

14기 공통 협업 세션 실습 레포지토리입니다.

이 저장소는 Git/GitHub 협업 흐름을 연습하기 위해 만들어졌습니다.  
Fork, Clone, Branch, Commit, Push, Pull Request 과정을 직접 실습합니다.

## 실습 목표

- 중앙 저장소와 개인 저장소의 차이를 이해합니다.
- 저장소를 fork하고 clone하는 흐름을 익힙니다.
- 새 브랜치를 만들어 작업합니다.
- 변경 사항을 commit하고 push합니다.
- Pull Request를 만들어 코드 리뷰 흐름을 경험합니다.

## 실습 순서

1. 이 저장소를 fork합니다.
2. fork한 저장소를 내 컴퓨터로 clone합니다.
3. 새 브랜치를 만듭니다.
4. `members/본인이름.md` 파일을 추가합니다.
5. 변경 사항을 commit합니다.
6. 내 fork 저장소에 push합니다.
7. 원본 저장소로 Pull Request를 보냅니다.

## 실습 과제

`members/example.md` 파일을 참고해서 `members/본인이름.md` 파일을 만들어 주세요.

파일에는 아래 내용을 포함합니다.

- 이름 또는 닉네임
- GitHub 아이디
- 한 줄 소개
- 오늘 실습에서 기대하는 점

## 브랜치 이름 예시

```bash
feature/add-my-profile
```

## 커밋 메시지 예시

```bash
docs: add my profile
```

## Pull Request 작성

Pull Request를 보낼 때는 자동으로 뜨는 템플릿을 채워 주세요.

관련 이슈가 있다면 PR 본문에 아래처럼 적을 수 있습니다.

```text
close #이슈번호
```
